1//! Interfaces and implementations specific to integration with `ndarray`.
2//!
3//! This module introduces the trait [`IntoNdArray`], which is implemented for
4//! all NIfTI volume types and enable their mapping into an [`Array`] with a
5//! dynamic number of dimensions and an abitrary element type. The affine
6//! scaling of the values (from the `scl_slope` and `scl_inter` attributes) are
7//! also considered in this transformation.
8//!
9//! A target [element type] needs to be provided at compile time, which is
10//! usually the data type that the user wishes to work the array with. If the
11//! source and target types do not match, each voxel is cast in a way as to
12//! avoid loss of precision.
13//!
14//! #### Note on memory order
15//!
16//! NIfTI volumes are usually stored in disk in column major order (also called
17//! Fortran order). As such, the array resulting from this operation will also
18//! be in this memory order, rather than the usual row major order (AKA C
19//! ordering). When accessing the array, one should consider any potential
20//! bottlenecks emerging from this ordering in their data processing pipelines.
21//! Namely, it might be faster to produce output arrays in column major order
22//! as well.
23//!
24//! [`IntoNdArray`]: ./trait.IntoNdArray.html
25//! [`Array`]: ../../../ndarray/type.Array.html
26//! [element type]: ../element/trait.DataElement.html
27//!
28use crate::error::Result;
29use crate::volume::element::DataElement;
30use crate::volume::NiftiVolume;
31use ndarray::{Array, Axis, Ix, IxDyn};
32
33/// Trait for volumes which can be converted to an ndarray.
34///
35/// Please see the [module-level documentation](index.html) for more details.
36pub trait IntoNdArray {
37    /// Consume the volume into an ndarray with the same number of dimensions
38    /// and the given target element type `T`.
39    fn into_ndarray<T>(self) -> Result<Array<T, IxDyn>>
40    where
41        T: DataElement;
42}
43
44impl<V> IntoNdArray for super::SliceView<V>
45where
46    V: NiftiVolume + IntoNdArray,
47{
48    fn into_ndarray<T>(self) -> Result<Array<T, IxDyn>>
49    where
50        T: DataElement,
51    {
52        // TODO(#94) optimize this implementation (we don't need the whole volume)
53        let volume = self.volume.into_ndarray()?;
54        Ok(volume.index_axis_move(Axis(self.axis as Ix), self.index as usize))
55    }
56}
