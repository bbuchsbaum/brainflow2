# Brainflow Memory Bank

This directory contains the consolidated project documentation for Brainflow, a high-performance neuroimaging visualization application.

## Quick Start

For new developers or returning after a break, read these files in order:

1. **Project_Blueprint.md** - Technical architecture, stack, and guidelines
2. **Project_Status.md** - Current progress, active work, and next steps

## Contents

### Active Documentation

- **Project_Blueprint.md**: Consolidated technical blueprint including:
  - Architecture overview and technology stack
  - Core components and patterns
  - Development guidelines and standards
  - Repository structure
  
- **Project_Status.md**: Current development status including:
  - Milestone progress (M1-M8)
  - Sprint details and task completion
  - Known issues and blockers
  - Next steps and priorities

### Archive

The `archive/` directory contains the original planning documents that were consolidated into the above files. These are preserved for historical reference and detailed specifications.

## Usage with AI Assistants

When working with AI coding assistants (Claude, Cursor, etc.), point them to this directory for project context. The consolidated files provide a complete overview without overwhelming token limits.

## Maintenance

- Update **Project_Status.md** regularly as work progresses
- Update **Project_Blueprint.md** when architecture decisions change
- Archive new planning documents that have served their purpose
